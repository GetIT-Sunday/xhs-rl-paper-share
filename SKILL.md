# 小红书 RL 论文自动化分享

自动完成：**抓取论文 → AI生成文案 → 截取封面 → 确认 → 发布**

## 使用场景

**适用**：
- 分享强化学习（RL）、具身智能、机器人学习领域的论文
- 自动生成小红书风格的论文解读文案
- 知识分享类账号的内容生产

**不适用**：
- 非RL领域论文（如CV、NLP纯应用）
- 技术新闻、产品发布、行业动态
- 需要深度实验复现的论文

## 核心能力

1. **论文抓取**：从 arXiv 抓取最新 RL 论文
2. **去重管理**：维护已发布论文列表，避免重复
3. **AI 深度生成文案**：理解论文内容后生成爆款文案（非模板填充）
4. **封面截取**：自动截取 arXiv 论文首页作为配图
5. **自动发布**：支持手动确认发布和定时发布

## 典型工作流

### 场景一：自动生成论文分享文案

用户说："帮我生成一篇RL论文的小红书文案"

1. 从已抓取的论文队列中选择一篇未发布的论文：
   ```bash
   python3 scripts/fetch_papers.py --count 5
   ```
2. 读取论文标题、摘要
3. **AI 深度理解论文内容**，参考 `references/xhs_style_guide.md` 和 `references/paper_template.md` 的风格要求，生成小红书文案
4. 展示给用户确认

### 场景二：完整自动化发布流程

用户说："自动发布一篇RL论文笔记" 或 "分享一篇RL论文"

1. 执行论文抓取脚本：
   ```bash
   python3 scripts/fetch_papers.py --count 5
   ```
2. 读取 `references/published_papers.json` 检查已发布列表
3. 选择一篇未发布的论文
4. **AI 生成文案**（见下方「AI 文案生成规则」）
5. 截取论文封面：
   ```bash
   python3 scripts/capture_cover.py --arxiv-id <id>
   ```
6. 展示预览（标题 + 正文 + 封面）给用户确认
7. 用户确认后执行发布：
   ```bash
   python3 scripts/publish_to_xhs.py --arxiv-id <id> --cookie '<cookie>' --title '<标题>' --desc '<正文>' --image <封面路径> --mark-published
   ```

### 场景三：定时发布

用户说："每天早上9点自动发布一篇论文笔记"

创建 infoflow_cron 定时任务，每天 9:00 触发，执行完整流程。

## AI 文案生成规则（核心）

**这是最重要的步骤，必须由 AI 深度理解论文后生成，禁止使用模板填充。**

### 输入

- 论文标题（英文）
- 论文摘要（英文）
- 论文 arXiv ID

### 生成要求

**标题**（标题党，20-30字）：
- 必须用中文
- 必须吸引眼球，传递价值
- ✅ "这篇RL综述太全了！强化学习入门必看"
- ✅ "具身智能+RL，这个方向要火了"
- ✅ "PPO/SAC/DQN全讲透了！RL算法全景解读"
- ❌ "Reinforcement Learning: An Overview 论文解读"
- ❌ "强化学习方法综述"

**正文**（300-500字，口语化）：
- 第一句：引入，建立共鸣（"从最新论文里挖到一篇好文..."）
- 📕 + 论文标题（英文原标题）
- 正文段落：用通俗语言解释论文核心内容
  - 把英文术语翻译成中文+保留关键英文缩写
  - 核心观点用【】或「」强调（不要用 ** ** 等 markdown 语法，小红书不渲染）
  - 使用 emoji 分段（📕📗📘）
  - 避免大段学术术语堆砌
  - 把抽象概念用类比/举例说明
- 结尾：引导互动（"你觉得这个方向怎么样？评论区聊聊～"）

**话题标签**：
- 基础：#机器学习 #强化学习 #文献阅读
- 根据内容添加：#具身智能 #多智能体 #离线强化学习 #世界模型 等

### 参考文案（必须模仿这个风格）

```
分享两篇RL综述，强化学习入门推荐！

从一站式具身智能RL入门指南里挖到两篇强化学习综述，非常适合强化学习入门，来分享一下！

📕《Reinforcement Learning: An Overview》
这篇文章全面综述了强化学习（RL）的核心方法、理论基础与前沿进展。文章系统梳理了基于价值（如Q-learning、DQN）、基于策略（如REINFORCE、A2C、PPO）、基于模型（如Dreamer、MuZero）三大类RL方法，并深入探讨了多智能体博弈、探索-利用权衡、稀疏奖励、离线学习、世界模型、表示学习、层次强化学习等关键问题。此外，文章还重点分析了RL与大模型（如LLM）结合的新趋势，以及基于概率推断的"控制即推理"框架（如SAC、MPO、主动推断）。整篇综述不仅覆盖了从表格型到深度强化学习的完整演进路径，也揭示了RL在样本效率、泛化能力与实际部署中的挑战与未来方向，是理解现代强化学习体系与前沿研究的指南之一。

#机器学习 #强化学习 #具身智能 #文献阅读
```

### 生成步骤

1. 阅读论文标题和摘要，理解论文核心内容
2. 判断论文类型：综述/方法/应用
3. 提取核心贡献、关键方法、实验结论
4. 将英文内容翻译为通俗中文
5. 按照上述格式组织文案
6. 确保标题吸引眼球、正文通俗易懂、标签覆盖关键词

## 资源说明

### scripts/

- `fetch_papers.py`：从 arXiv 抓取最新 RL 论文
- `capture_cover.py`：截取论文 arXiv 首页作为封面图
- `publish_to_xhs.py`：发布到小红书（支持标题/正文/图片/私密/定时/标记已发布）

### references/

- `published_papers.json`：已发布论文列表（避免重复）
- `paper_template.md`：文案模板参考（AI生成时的风格参考）
- `xhs_style_guide.md`：小红书爆款文案风格指南

### assets/

- `covers/`：论文封面图存储目录

## Cookie 管理

发布需要小红书 Cookie，从浏览器获取：
- `a1`
- `web_session`
- `webId`

Cookie 存储位置：`~/.openclaw/workspace/.xhs_cookie`

## 错误处理

| 错误类别 | 识别特征 | 处理原则 |
|-|-|-|
| Cookie 失效 | 401/403 | 停止执行，提示重新获取 Cookie |
| 论文不存在 | arXiv 404 | 跳过该论文，选择下一篇 |
| 图片上传失败 | 网络/格式错误 | 重试 1 次，仍失败则跳过 |
| 发布失败 | API 错误 | 保存文案到本地，提示手动发布 |
| 重复论文 | 已在发布列表 | 跳过，选择下一篇未发布的 |
